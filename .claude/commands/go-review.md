# Go Code Review

## 対象ファイル

$ARGUMENTS

## レビュー手順

1. 対象ファイルを読み込む
2. go-code-review スキルに基づいてレビュー
3. 以下の観点で問題点と改善案を提示

## レビュー観点

### 必須チェック項目
- [ ] エラーは適切に処理されているか
- [ ] リソース（ファイル、接続）は適切にクローズされているか
- [ ] 並行処理は安全か（データ競合、goroutine リーク）
- [ ] 入力検証は行われているか

### コードスタイル
- [ ] 命名は明確か
- [ ] 関数は適切な長さか
- [ ] 早期リターンを使っているか
- [ ] コメントは適切か

### パフォーマンス
- [ ] 不要なメモリ割り当てはないか
- [ ] 大きな構造体はポインタで渡しているか

### セキュリティ
- [ ] SQL インジェクション対策
- [ ] 機密情報のログ出力チェック

## 出力フォーマット

```markdown
## レビュー結果: [ファイル名]

### 良い点
- ...

### 改善が必要な点

#### 1. [問題の概要] (優先度: 高/中/低)
**場所**: 行 XX
**問題**: ...
**改善案**:
```go
// 改善後のコード
```

### 全体的なコメント
...
```
